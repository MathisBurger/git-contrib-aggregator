# Contributing to git-contrib-aggregator

Thank you for considering contributing to **git-contrib-aggregator**! Your contributions help keep this project useful, lightweight, and easy to maintain.


## How to Contribute

### Reporting Issues

* If you find a bug or have a feature request, please open an issue.
* Provide a clear description of the problem and steps to reproduce it.
* Include any relevant environment information, like your GitLab or GitHub API version.

### Submitting Pull Requests

1. Fork the repository.
2. Create a new branch for your feature or bugfix:

   ```bash
   git checkout -b my-feature
   ```
3. Make your changes.
4. Ensure the code is formatted using `cargo fmt` and builds without errors.
5. Add or update tests if applicable.
6. Commit your changes with a clear message.
7. Push your branch and open a pull request.

### Code Style

* Use **Rust 2021 edition**.
* Format code with `cargo fmt`.
* Keep the project lightweight: avoid adding niche or overly complex features.

### Testing

* Ensure that any new functionality is tested.
* Existing tests should continue to pass.


## Guidelines

* Contributions should focus on **general-purpose improvements** and essential features.
* Avoid adding unnecessary bloat.
* Respect the philosophy of the project: **simple, minimal, and reliable**.


## Communication

* Feel free to discuss ideas, questions, or clarifications in the issues section.
* Be respectful and constructive in all interactions.


Thank you for helping make **git-contrib-aggregator** better!
